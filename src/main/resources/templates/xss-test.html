XSS 보호 테스트 페이지

XSS 보호 정보:

이 페이지에서는 애플리케이션에 구현된 XSS 테스트할 수 있습니다.

테스트 1: 쿼리 파라미터 XSS 보호

GET 요청을 사용하여 쿼리 파라미터에 대한 XSS 보호를 테스트합니다.

Original Input
아직 데이터가 없습니다...
Sanitized Output
아직 데이터가 없습니다...

테스트 예제 (클릭하여 사용):

<script>alert('XSS')</script> <img src="x" onerror="alert('XSS')"> <a href="javascript:alert('XSS')">Click me</a> <iframe src="javascript:alert('XSS')"></iframe> <svg onload="alert('XSS')"> Normal text without XSS

테스트 2: 요청 본문 XSS 보호

POST 요청을 사용하여 JSON 요청 본문에 대한 XSS 보호를 테스트합니다.

아직 데이터가 없습니다...
Sanitized Output
아직 데이터가 없습니다...

테스트 예제 (클릭하여 사용):

<script>alert('XSS')</script> <img src="x" onerror="alert('XSS')"> <a href="javascript:alert('XSS')">Click me</a> <iframe src="javascript:alert('XSS')"></iframe> <svg onload="alert('XSS')"> Normal text without XSS

테스트 3: HTML 콘텐츠 XSS 보호

사용자 입력이 포함된 HTML 콘텐츠를 반환할 때 XSS 보호를 테스트합니다.

Original Input
아직 데이터가 없습니다...
Sanitized Output
아직 데이터가 없습니다...

테스트 예제 (클릭하여 사용):

<script>alert('XSS')</script> <img src="x" onerror="alert('XSS')"> <a href="javascript:alert('XSS')">Click me</a> <iframe src="javascript:alert('XSS')"></iframe> <svg onload="alert('XSS')"> Normal text without XSS <div style="color: red">Styled text</div>

이 테스트 페이지 사용 방법

  1. 테스트 선택: 위의 세 가지 테스트 섹션 중 하나를 선택하세요.
  2. 입력 입력: 직접 입력하거나 예제 페이로드 중 하나를 클릭하세요.
  3. 테스트 실행: 테스트 버튼을 클릭하여 XSS 보호가 입력을 어떻게 처리하는지 확인하세요.
  4. 결과 비교: 원본 입력과 Satilized된 출력이 나란히 표시됩니다.
  5. Iframe 확인: HTML 콘텐츠 테스트의 경우, iframe은 콘텐츠가 브라우저에서 어떻게 렌더링되는지 보여줍니다.

참고: 이 페이지는 테스트 목적으로만 제공됩니다.
